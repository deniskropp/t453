⫻context/klmx:RefinedEventSchemas/1
```
schema event {
  type: "decision" | "metrics" | "memory_write" | "arbitration" | "visual"
  payload: any
  timestamp: int
  source_stage: string
  correlation_id: string
}

schema decision_event {
  extends event
  type: "decision"
  payload: {
    consent: "granted" | "conditional" | "denied"
    scope: string[]          // runtime-enforceable operational guard
    expiry: int?             // cycle count or absolute timestamp — runtime guard
    rationale: string
    arbitration_required: bool
  }
}

schema metrics_event {
  extends event
  type: "metrics"
  payload: {
    coherence_score: float
    novelty_delta: float
    consent_stability: float
    cycle: int
  }
}

schema memory_write_event {
  extends event
  type: "memory_write"
  payload: {
    memory_type: "episodic" | "semantic" | "policy"
    write_policy: string     // e.g. "append-only", "validated-merge", "immutable-log"
    content: any
  }
}
```

⫻context/klmx:MemoryWritePolicies/1
Stage20 enforces strict separation via write policies (prevents raw trace pollution of validated knowledge):

- episodic: append-only, immutable, full TAS traces + embodiment signals + raw co-creation events. Never mutated.
- semantic: validated-merge only. New patterns accepted only after coherence + consent checks; duplicates or low-novelty items rejected or versioned.
- policy: immutable-log. Every arbitration outcome, consent decision, Guard evolution, and hard-stop is appended with full rationale and correlation_id. Never overwritten.

These policies are declared in the Meta-DNA and enforced at write time.

⫻context/klmx:RuntimeContract/1
Portable contract for any implementation (KickLang native, OCS, TypeScript event-sourced):

- Stage10 always emits a typed decision_event. Scope and expiry are runtime-enforced guards (execution may be truncated or scoped at runtime).
- Stage16 performs arbitration only when decision_event.arbitration_required == true. Declared strategy is mandatory.
- Stage20 writes exactly three memory streams via memory_write_event, each obeying its declared write_policy.
- Stage00 loads only memory slices whose scope overlaps current intent/decision scope (prevents unnecessary hydration).
- All metrics_event records are appended to an immutable event log. Convergence is computed deterministically from the log:
  ```
  convergence := latest_metrics.coherence_score > 0.9
              AND latest_metrics.novelty_delta < 0.05
              AND latest_metrics.consent_stability > 0.95
  ```
- Every event carries correlation_id for full replay and audit across cycles.
- Meta-DNA block is emitted at Stage20 and can be versioned or transplanted.

This contract guarantees unambiguous implementation and preserves the self-describing Meta-DNA property.
